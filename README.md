# InsightAgent

**AI-Powered E-Commerce KPI Investigation System**

InsightAgent is a multi-agent AI system that investigates business KPI anomalies and performs root-cause analysis on e-commerce data. Ask a natural-language business question - like *"Why are customer review scores low?"* - and get back an evidence-backed executive report, generated by a pipeline of specialized AI agents working in parallel.

**[Live Demo](#)** - https://insightagent-xtbumcy6rji2buqtkjgqy6.streamlit.app/

---

## What It Does

Given a business question, InsightAgent:
1. **Classifies** which business domains are relevant (customer satisfaction, delivery, seller performance, product category)
2. **Investigates** each relevant domain in parallel, computing real statistics from 112,650+ real e-commerce transactions
3. **Ranks root causes** by evidence strength using a deterministic severity score - not guesswork
4. **Generates an executive report** using an LLM, but only for writing - every number is traced back to real, auditable computation

Built with **LangGraph** - the Planner uses keyword-based routing to decide which of the 4 analysis agents to call; only relevant agents run, in parallel, via LangGraph's conditional edge and state-merging capabilities.

## Key Design Decision: LLM for Writing, Not Computing

All quantitative analysis (review score averages, delay rates, seller/category rankings) is done in deterministic pandas code - **never** by the LLM. The LLM (Groq's LLaMA 3.3 70B) is used exclusively in the final Report Agent, to convert already-correct structured findings into polished, readable prose. This ensures every number in the final report is accurate, reproducible, and traceable — not hallucinated.

## Tech Stack

- **Orchestration:** LangGraph (StateGraph, conditional edges, parallel execution)
- **LLM:** Groq API (LLaMA 3.3 70B Versatile)
- **Data:** Pandas, on the [Olist Brazilian E-Commerce dataset](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce) (112,650 order-item rows)
- **Frontend:** Streamlit
- **Environment:** Python, python-dotenv for secrets management

## Sample Finding

One of the strongest insights the system surfaces automatically:

> Delayed orders average **2.55 stars**, compared to **4.21 stars** for on-time orders - a **1.66-point gap**, making delivery delay the single strongest driver of customer dissatisfaction in the dataset.

## Known Limitations

- Root cause synthesis currently includes all findings from triggered agents rather than filtering strictly by question intent - occasionally surfaces tangential causes
- Severity ranking is globally consistent across cause types but not question-aware, so a platform-wide metric can outrank a more directly relevant category-specific one
- Planner uses keyword-based routing (not LLM-based) - fast, free, and fully explainable, but less flexible to entirely novel phrasings outside the tested question set
